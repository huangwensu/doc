1.基础知识
  （1）算法
  （2）数据结构
  （3）面向对象思想
  （4）设计模式

2.第一个标准ECMA-262标准
  （1）原型和原型链
  （2）作用域和闭包
  （3）异步
  （4）ES6的新增语法（如模块化、class、promise等）

3.第二个标准W3C标准
  （1）HTML
  （2）CSS
  （3）Javascript Web API （例如 DOM 操作、BOM 操作、事件绑定、ajax 等）
  （4）HTTP 协议
  （5）存储
  （6）websocket
  （7）JSON 和 XML
  （8）2D 3D
    标准一方面指导开发人员如何写代码，一方面指导浏览器如何运行代码。即，代码和浏览器之间，就靠这个标准来紧密相连。

4.开发环境
  （1）代码版本管理，如 git
  （2）IDE ，如 sublime vscode vim 等
  （3）脚手架，如 vue-cli
  （4）构建工具，如 webpack
  （5）本地服务和 mock ，如 webpack-dev-server
  （6）debug ，如用到 chrome 开发者工具、抓包、代理等
  （7）部署提测，各个公司都不一样，基本原理都是将代码压缩然后上传到测试机
  （8）单元测试
  （9）有时需要操作测试机，需要掌握 linux 基础命令

5.运行环境（标准浏览器运行）
       （1）加载和渲染过程，如面试中常被问的“请描述从输入 url 到呈现出页面的整个过程”
       （2） 性能优化，如何更快的加载，如何更快的解析和渲染
       （3）安全问题，常见的有 XSS 和 CSRF
       （4） 浏览器兼容性
       （5） 响应式布局

6.框架和库
  任何框架和库的出现，都是为了满足我们日常开发的效率和设计，说白了都是对已有方式的一种改进和补充（因为基础和标准不常变）
